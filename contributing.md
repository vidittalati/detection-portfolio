# Contributing to Detection Portfolio

Thank you for your interest in contributing to this detection portfolio.  
This repository is maintained as a showcase of practical, real-world detection engineering workflows including threat research, Sigma rule creation, SIEM translation, false-positive tuning, and log validation.

While this is a personal project, contributions that improve educational value, technical depth, or detection quality are welcome.

---

## 📌 Contribution Philosophy

This repository follows four core principles:

### 1. **Accuracy**
Detections must reflect realistic attacker behaviors and map correctly to MITRE ATT&CK TTPs, log sources, and fields.

### 2. **Clarity**
Every detection must include:
- clear rationale  
- detection logic  
- false-positive expectations  
- tuning guidance  
- relevant MITRE mappings  

### 3. **Reproducibility**
Any detection submitted should include:
- example logs  
- Sigma rule  
- SIEM query translation (Splunk preferred)  
- validation notes  

### 4. **Security First**
Contributed rules should prioritize behavioral detections over simple IOC lookups to avoid brittle or noisy detections.
